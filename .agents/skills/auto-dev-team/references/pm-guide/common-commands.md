# 常用指令速查

> 💡 这些是 PM 常用的指令，AI 会识别并执行相应操作。

## 控制指令

| 说法 | AI 行为 |
|------|---------|
| "确认" / "ok" / "继续" | 进入下一步 |
| "回退" / "撤销" | 执行读档流程，展示存档列表让用户选择 |
| "回退到「xxx」" | 回退到指定指纹的存档 |
| "重新来" / "换个方案" | 放弃当前计划，重新分析 |
| "暂停" / "先停一下" | 保存当前状态，等待指示 |
| "不对，我要的是..." | 理解新需求，调整方向 |

## 模式切换

| 说法 | 进入模式 |
|------|----------|
| "我想做个新功能" | Architect |
| "这里有 bug" / "报错了" | Debug |
| "线上挂了！紧急！" | Hotfix |
| "代码太乱了，整理一下" | Refactor |
| "这个太慢了" | Optimize |
| "帮我看看这个项目" | Survey |
| "删掉没用的代码" | Cleanup |
| "加个测试" / "帮我出预发验收单" / "根据最近提交列测试数据" / "先给我查预发数据 SQL" / "给我一份发版手测步骤" | Tester |
| "这段代码什么意思" | Explain |
| "改个文案" / "调个颜色" | FastTrack |

## 执行控制

| 说法 | AI 行为 |
|------|---------|
| "信任模式" / "连续执行" | 连续执行多步，完成后统一汇报 |
| "一步一步来" | 每步都等确认（默认） |
| "详细说明" | 展开更多解释 |
| "直接执行" / "不用解释" | 跳过详细说明 |
| "执行方案" / "写代码" / "开始" | 💿 自动触发执行前快照闸门，然后开始写代码 |

## 验收相关

| 说法 | AI 行为 |
|------|---------|
| "好了" / "没问题了" | 确认完成，进入收尾 |
| "还有问题" | 继续调试/修改 |
| "验收一下" | 展示验收清单 |
| "我信任结果" | 跳过验收，直接结束 |

## 分支与合并

| 说法 | AI 行为 |
|------|---------|
| "合并" / "合并到主分支" | 将工作分支合并到集成分支 |
| "推送" / "push" | 推送到远程仓库 |
| "合并并推送" | 合并到集成分支并推送 |
| "只推工作分支" | 推送工作分支（便于创建 PR） |
| "创建 PR" | 推送工作分支，提示去创建 PR |
| "当前在哪个分支" | 显示当前分支信息 |

## 信息查询

| 说法 | AI 行为 |
|------|---------|
| "现在做到哪了" | 展示当前进度 |
| "之前改了什么" | 展示改动历史 |
| "能回退吗" / "有哪些存档" | 执行读档流程，展示存档列表（基于真实 git 数据） |
| "建立里程碑" | 建立 🎯 里程碑（commit + tag） |
| "这个项目结构是什么" | 展示 project-map 摘要 |

## 紧急情况

| 说法 | AI 行为 |
|------|---------|
| "停！" / "别动了！" | 立即停止所有操作 |
| "全部回退" / "回到最开始" | 执行读档流程，高亮里程碑选项 |
| "出问题了" / "代码写坏了" | 确认是否需要回退，是则执行读档流程 |
