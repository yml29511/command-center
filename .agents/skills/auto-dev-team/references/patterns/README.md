
## 三层分类说明

| 层级 | 目录 | 内容 | 示例 |
|------|------|------|------|
| **通用原理** | `universal/` | 语言无关的设计思想 | 排队机制、缓存策略、重试模式 |
| **语言经验** | `language/` | 特定语言的陷阱和技巧 | TS 的类型体操、Python 的 GIL |
| **平台经验** | `platform/` | 特定平台的最佳实践 | Android ANR、iOS 内存管理 |

## AI 检索逻辑

```
当任务明确需要复用经验时:
1. 识别任务领域（并发/性能/错误处理...）
2. 识别项目语言和平台
3. 按顺序搜索:
   → universal/{领域}/     # 先看通用原理
   → language/{语言}/      # 再看语言经验
   → platform/{平台}/      # 最后看平台经验
4. 综合后给出建议
```

```markdown

> 一句话：什么场景用这个 pattern

| 维度 | 范围 |
|------|------|
| 语言 | 通用 |
| 平台 | 通用 |

## 问题

[用自然语言描述问题，不涉及具体语言]

[用伪代码或流程图描述核心思路]
```

## 如何新增 Pattern

### 触发条件

任务完成后，AI 会判断：
1. 是否解决了有代表性的问题？
2. 方案是否经过验证？
3. 能否复用到其他项目？

### 分类决策

```
这个经验是...
├── 语言无关的通用思想？ → universal/
├── 某语言特有的陷阱/技巧？ → language/{语言}/
└── 某平台特有的最佳实践？ → platform/{平台}/
```

### 命名规范

- 文件名: `kebab-case.md`（如 `token-refresh.md`）
- 避免使用特定业务名称
- 使用动词或名词短语描述行为
