## 核心概念

| 术语 | 图标 | 定义 | 存储 |
|------|------|------|------|
| **里程碑** | 🎯 | 核心版本锚点，标记任务边界 | commit + git tag |
| **保护快照** | 💿 | 写代码前的自动现场保护 | commit |
| **存档** | 💾 | 每步改动完成后的记录 | commit |
| **指纹** | - | ≤10 字的业务功能摘要 | commit message |
| **工作分支** | - | AI 干活用的本地分支，与主分支隔离 | - |
| **受保护分支** | - | 不允许直接操作的分支（如 main/master/production） | - |
| **集成分支** | - | 功能完成后要合并到的目标分支 | - |

## 配置来源

- `.autodev/autodev-config.json`
  - skill 行为策略：保护分支、会诊阈值、版本保护默认行为
- `.autodev/path.md`
  - 项目环境、Git 事实、部署入口、集成分支约定

默认优先级：

1. 项目显式配置
2. `origin/HEAD`
3. 当前分支 / skill 默认值

## 脚本接口（优先）

| 动作 | 命令 |
|------|------|
| 分支守卫 | `scripts/checkpoint.sh ensure-branch <task-slug>` |
| 建立里程碑 | `scripts/checkpoint.sh milestone "<指纹>" "<描述>" <task-slug>` |
| 执行前快照闸门 | `scripts/checkpoint.sh snapshot-gate <task>` |
| 建立存档 | `scripts/checkpoint.sh archive "<指纹>" <type> "<描述>"` |
| 展示存档列表 | `scripts/checkpoint.sh list` |
| 回退到目标 | `scripts/checkpoint.sh rollback <hash>` |

脚本不可用时，再按本文手工兜底。

## 分支安全规则

### 1. 受保护分支

默认受保护分支：

- `main`
- `master`
- `production`
- `release/*`

### 2. 集成分支（合并目标）

优先级：

1. `.autodev/path.md` 中的 `integration_branch`
2. `origin/HEAD` 指向的分支
3. 任务起始时所在分支

### 3. 工作分支命名

| `integration_mode` | 格式 | 示例 |
|--------------------|------|------|
| `merge_allowed` | `autodev/<任务简述>-<MMDDHHmm>` | `autodev/会员登录-03071430` |
| `pr_only` | `autodev/<actor>/<任务简述>-<MMDDHHmm>` | `autodev/rex/会员登录-03071430` |

`actor` 取自 `git config user.name` 的首段，统一转成小写 slug。

### 4. 任务启动时的分支守卫

AI 开始任何代码改动前，**必须执行**：

```text
检查当前分支
├─ 在受保护分支 → 自动创建工作分支，输出提示
└─ 在工作分支 → 继续
```

创建工作分支后输出：

```text
━━━━━━━━━━━━━━━━━━━━
🌿 已创建工作分支
分支: autodev/会员登录-03071430
基于: main (a1b2c3d)
━━━━━━━━━━━━━━━━━━━━
```

## 🎯 里程碑

里程碑代表“可信赖的安全回退点”。保持稀缺，一个任务通常只有 1-2 个。

### 触发条件（只有 4 种）

| # | 触发点 | 条件 |
|---|--------|------|
| 1 | **任务开始** | 进入任何写入模式时，在 write-preflight 中建立 |
| 2 | **信任模式开始** | 用户选择信任模式执行前 |
| 3 | **任务完成** | 用户确认任务完成时（可选，询问用户） |
| 4 | **用户指定** | 用户说“建立里程碑” |

### 指纹规范

- `{业务摘要}#起点`
- `{业务摘要}#信任起点`
- `{业务摘要}#完成`
- `{用户指定名}`

### 提交规则

- `git add -A`
- `git commit --allow-empty -m "「{指纹}」chore: {描述}"`
- `git tag milestone/{任务简述}-{MMDDHHmm}`

`pr_only` 模式下 tag 格式为 `milestone/{actor}/{任务简述}-{MMDDHHmm}`。

### 输出格式

```text
━━━━━━━━━━━━━━━━━━━━
🎯 里程碑
━━━━━━━━━━━━━━━━━━━━
指纹: {任务}#起点
哈希: c3d4e5f
标签: milestone/{任务}-{MMDDHHmm}
━━━━━━━━━━━━━━━━━━━━
💡 随时可说「回退到 {任务}#起点」
━━━━━━━━━━━━━━━━━━━━
```

## 💿 保护快照

保护快照是写代码前的安全网，保留“改动前”状态。分为**执行前闸门（强制）**和**补充快照（智能触发）**两层。

### 执行前快照闸门（强制）

任何实际代码写入前，必须通过闸门。

**⛔ Fail-Close 规则**：若未输出 `💿 已保护` 或 `💿 闸门通过`，禁止执行任何文件写入操作。

#### 触发时机

| 场景 | 触发点 |
|------|--------|
| Step 模式 | 每个 Step 开始写代码前 |
| 信任模式 | 🎯 里程碑之后、首次写入前 |
| FastTrack | 用户确认方案后、改动前 |
| Hotfix | 用户确认方案后、修复前 |
| Optimize 单点 | 用户确认方案后、优化前 |
| Debug 治疗 | 用户确认方案后、执行前 |
| Cleanup | 用户确认范围后、清理前 |
| Tester | 确认测试方案后、写测试前 |

#### 闸门逻辑

```text
执行指令到达
├─ 有未提交改动 → 建立 💿 快照 commit
└─ 工作区干净
    ├─ 已有当前任务基线（🎯 / 💿 / 💾） → 输出"闸门通过"
    └─ 无当前任务基线 → 建立允许为空的 💿 快照 commit
```

#### 闸门输出

```text
💿 已保护 → a1b2c3d（执行前闸门）
```

或：

```text
💿 闸门通过（基线 a1b2c3d 即保护点）
```

### 补充快照（智能触发）

满足任一条件时建立补充快照：

1. 距离上次 commit > 10 分钟且即将写入
2. 即将改动的文件与上次 commit 无关（跨模块）

提交格式：`git commit --allow-empty -m "「{任务}#保护」chore: 自动存档"`

## 💾 存档

存档是每步改动后的细粒度记录，用于逐步回退。

### 各模式存档策略

| 模式 | 触发时机 | 指纹格式 |
|------|----------|----------|
| Step 模式 | 每步验证通过后 | `「{任务}#{序号}」` |
| 信任模式 | 全部完成后（1 个） | `「{任务}#信任完成」` |
| FastTrack | 改动验证通过后 | `「{改动}#01」` |
| Debug / Hotfix | 修复验证通过后 | `「{问题}#修复」` / `「HOTFIX#{问题}」` |
| Refactor / Optimize | 每步等价 / 性能验证后 | `「{目标}#{序号}」` |
| Cleanup | 清理验证后 | `「清理#{日期}」` |
| Tester | 测试通过后 | `「{测试}#01」` |

### 指纹规范

- 长度：≤ 10 个字
- 内容：业务功能摘要，描述“做了什么功能”
- 唯一性：`<业务摘要>#<两位序号>`

### Commit 格式

`git add -A && git commit -m "「{指纹}」{类型}: {技术描述}"`

类型：`feat` | `fix` | `refactor` | `perf` | `docs` | `chore`

### 暂存区审查

每次 `git add -A` 之后、`git commit` 之前，必须执行：

```bash
git diff --cached --name-only
```

以下模式的文件必须重点审查：

```text
.autodev/**
.cursor/**
.env / .env.*
*.pem / *.key / *.p12
credentials.* / secrets.*
```

| `integration_mode` | 处理方式 |
|--------------------|----------|
| `merge_allowed` | ⚠️ 警告并列出文件，默认不自动移除 |
| `pr_only` | ⛔ 自动执行 `git reset HEAD <file>` 移出后再提交 |

> 里程碑和保护快照同样执行审查，但仅针对敏感文件（`.env`、密钥类），不过滤 `.autodev/`。

### 输出格式

```text
💾【存档】{任务}#{序号} → b2c3d4e
```

## 读档（展示存档列表）

读档必须基于真实 git 数据。

### 触发词

- “回退” / “撤销”
- “能回退吗” / “有哪些存档”
- “出问题了” / “代码写坏了”
- “全部撤销” / “回到最开始”

### 执行步骤（强制）

1. 执行 git 命令获取真实数据：

```bash
git log --oneline -20
git tag -l "milestone/*" --sort=-creatordate
```

2. 对每条 commit 判断类型：

```text
git tag --points-at {hash} 含 milestone/* → 🎯 里程碑
指纹含 #保护                            → 💿 保护快照
其他                                     → 💾 存档
```

3. 默认展示：
   - 所有里程碑
   - 最近 1 个保护快照
   - 最近 3 个存档

### 输出格式

```text
━━━━━━━━━━━━━━━━━━━━
📍 存档列表
━━━━━━━━━━━━━━━━━━━━
[1] 🎯 a1b2c3d「会员登录#起点」
[2] 💿 b2c3d4e「会员登录#保护」
[3] 💾 c3d4e5f「会员登录#01」
[4] 💾 d4e5f6g「会员登录#02」
━━━━━━━━━━━━━━━━━━━━
回退到哪个？（输入序号或指纹）
━━━━━━━━━━━━━━━━━━━━
```

## 回退执行

### 回退方式

- “回退到「会员注册表单#01」”
- “回退到 b2c3d4e”
- “回退”

### 回退安全检查

```text
回退请求
├─ 当前在工作分支
│   ├─ 工作区干净 → 执行回退
│   └─ 工作区有未提交改动 → 先建保护快照「现场保存#NN」，再回退
└─ 当前在受保护分支 → 拒绝，提示切换分支
```

### 执行回退

`git reset --hard {hash}`

### 回退输出

```text
━━━━━━━━━━━━━━━━━━━━
✅ 已回退
当前位置:「{指纹}」{hash}
分支: {工作分支}
━━━━━━━━━━━━━━━━━━━━
```

## 里程碑：合并与推送询问

以下情况必须询问用户是否合并 / 推送：

1. 信任模式任务完成后
2. Step 模式完成一个完整功能后
3. 用户选择“结束”时
4. AI 感知到阶段性成果完成时

### 合并执行原则

- `merge_allowed`
  - 可合并到集成分支
  - 默认使用 squash merge
  - 推送前必须再向用户确认
- `pr_only`
  - 不允许本地 merge 到集成分支
  - 只允许推送工作分支，供 GitHub / GitLab 创建 PR

## 默认策略

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `push_default` | `false` | 不自动推送，需用户明确选择 |
| `merge_strategy` | `squash` | 合并时 squash，保持主线整洁 |
| `integration_mode` | `merge_allowed` | 允许本地合并；`pr_only` 则只推工作分支、拒绝本地 merge 到集成分支 |

### `integration_mode` 行为对照

| 行为 | `merge_allowed` | `pr_only` |
|------|-----------------|-----------|
| 本地合并到集成分支 | ✅ 允许 | ⛔ 拒绝 |
| 推送工作分支 | ✅ 允许 | ✅ 允许 |
| 暂存区审查级别 | warn（警告，可继续） | block（阻断，自动移出） |
| Milestone tag 清理 | 关闭 | 自动（按任务前缀保留最近 2 个） |
| 分支 / tag 命名含 actor | 否 | 是 |

## Milestone Tag 清理

仅在 `pr_only` 模式下，任务完成并推送工作分支后执行。

清理规则：

1. 按当前任务的 tag 前缀匹配：`git tag -l "milestone/{任务简述}*" --sort=-creatordate`
2. 保留最近 2 个
3. 删除更早的 tag：`git tag -d {旧tag}`

⛔ 仅清理当前任务前缀的 tag，不触碰其他任务的 milestone。

## 异常处理

| 情况 | 处理方式 |
|------|----------|
| 无 `.autodev/path.md` | 提示用户先创建，使用模板 `assets/templates/path.md` |
| 当前在受保护分支 | 自动创建工作分支后再建立里程碑 |
| push 失败 | 报告错误，保留本地 commit，提示手动处理 |
| 冲突 | 停止自动流程，提示用户解决冲突后手动处理 |
| 无改动 | 跳过 commit，正常输出任务完成报告 |

## 禁止行为

- ❌ 在受保护分支上直接建立存档 / 里程碑
- ❌ 不询问用户就自动合并到集成分支
- ❌ 不询问用户就自动推送到远程
- ❌ 在受保护分支上执行 `git reset --hard`
- ❌ 回退时不保存工作区未提交改动
- ❌ 不执行 git 命令就凭记忆展示存档列表
- ❌ 编造不存在的 commit hash
- ❌ `pr_only` 模式下执行本地 merge 到集成分支
- ❌ 跳过暂存区审查直接 commit
