# 验收清单

> 💡 可选使用。任务完成时说"验收一下"可查看，说"我信任结果"可跳过。

## PM 验收

- [ ] **核心功能**: 描述的功能是否可用？
- [ ] **边界情况**: 空数据、错误输入时表现正常？
- [ ] **关键流程**: 页面跳转、提交、保存、权限等用户链路是否正常？
- [ ] **用户体验**: 操作是否顺畅，有无卡顿或明显异常？

## AI 自动验收

- [ ] 无 lint 错误
- [ ] 后台自动测试已执行，并有命令 / 输出证据
- [ ] GUI 自治验收状态明确（未触发 / 已通过 / 用户禁用 / 暂不可执行 / Manual only）
- [ ] GUI-capable task 已记录 executor、证据与 Gate 结论
- [ ] 待业务确认项已记录并说明状态
- [ ] 剩余风险已说明
- [ ] 文档已更新（project-map / module-registry）
- [ ] 存档已建立（工作分支隔离，含指纹）
- [ ] 里程碑询问已完成（是否合并/推送）

## 回归验收

- [ ] 已有功能未受影响
- [ ] 相关页面/功能正常工作

---

## 快速验收流程

### 小测试
```
AI: "✅ 已完成，并已输出 🧾 测试回执"

PM 选择:
[1] 看一下效果 → 手动验证
[2] 结束 → 信任结果
```

### 大测试 / 新功能开发
```
AI: "✅ 任务完成，测试台账见 .autodev/current-test.md"

PM 选择:
[1] 验收清单 → 逐项检查
[2] 快速验收 → 只测核心功能和关键流程
[3] 查看测试台账 → 看覆盖场景、待确认项、剩余风险
[4] 结束 → 信任结果
```

### Bug 修复
```
AI: "✅ Bug 已修复，并已输出 🧾 测试回执"

PM 必须验证:
1. 用原复现步骤测试 → 问题应该消失
2. 相关功能正常 → 没有引入新问题
3. 若命中 GUI-capable task → 检查 GUI 自治验收是否已通过
```
