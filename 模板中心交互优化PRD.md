## 1. 项目背景与目标

### 1.1 项目背景

本项目是一个「指令中心 / 模板中心」管理系统，面向 AI 指令模板的全生命周期管理，涵盖指令的创建、编辑、调试和运行监控。系统目前包含两个核心页面：

- **指令列表页（模板中心）**：展示所有指令模板，支持基本的增删改查操作。
- **指令调试页**：提供 Prompt 编辑与实时调试运行能力。

随着指令数量和业务场景的扩展，现有页面在信息组织、操作效率和调试体验方面逐渐暴露出不足，亟需通过交互优化提升整体可用性。

### 1.2 优化目标

1. **提升信息密度**：在列表页更合理地展示指令的关键元信息，减少无效留白。
2. **增强筛选能力**：引入分组筛选机制，帮助用户快速定位目标指令。
3. **精简调试界面**：将调试页聚焦于 Prompt 编辑这一核心任务，移除非必要字段的干扰。
4. **完善运行可观测性**：新增运行记录抽屉和操作日志弹窗，完整呈现调试执行历史与浏览器操作细节。
5. **统一视觉规范**：建立设计系统，确保页面视觉语言的一致性。

## 2. 现状问题分析

### 2.1 列表页问题

#### 问题一：信息密度不足

**现状**：卡片仅展示指令名称和描述片段，缺少分组归属、管理员、权限等关键元信息。用户无法一眼识别指令的业务属性。

**截图说明**：列表页截图显示每张卡片上方有大面积蓝色渐变占位区域（齿轮图标），卡片实际可用于展示信息的面积被大幅压缩。

#### 问题二：布局利用率低

**现状**：采用 4 列网格卡片布局，卡片上方存在大面积视觉占位图，有效信息区域占比低。同一行挤入过多卡片，导致单卡宽度受限。

**截图说明**：原列表页截图可见卡片内大量空间被顶部蓝色渐变区域和齿轮图标占据，名称和描述仅能展示极短内容。

#### 问题三：搜索筛选单一

**现状**：搜索栏仅有「名称」单一输入框，当指令数量增多时，用户无法通过分组等维度快速过滤。

**截图说明**：搜索区域仅包含一个名称输入框，无分组、状态等其他筛选条件。

#### 问题四：操作入口层次不清晰

**现状**：卡片底部平铺「编辑」「另存为模板」「删除」三个操作按钮，高频与低频操作未做区分，视觉上过于密集。

**截图说明**：卡片底部横向排列多个文字按钮，彼此间距紧凑，缺乏视觉优先级区分。

#### 问题五：分组管理缺失

**现状**：页面没有分组概念，所有指令平铺展示。用户无法按业务分组进行组织和浏览。

**截图说明**：页面直接展示卡片网格，无任何分组标题或归类标识。

### 2.2 调试页问题

#### 问题一：编辑区信息混杂

**现状**：左侧编辑区域混合展示指令名称、Prompt 内容、封面图 URL 输入框、变量表格等多种信息，视觉分层不清晰，用户难以聚焦。

**截图说明**：调试页截图显示左侧单张卡片内包含名称输入框、沙箱类型选择、Prompt 编辑区、封面 URL、变量表格等，内容堆叠严重。

#### 问题二：非核心字段挤占编辑空间

**现状**：调试时强制展示封面 URL 输入框和变量表格，这些非调试必需字段占用了宝贵的纵向空间，Prompt 编辑器面积被严重压缩。

**截图说明**：Prompt textarea 实际可用高度被上下方的表单字段挤压，长文本编辑体验差。

#### 问题三：运行记录缺失

**现状**：调试页无法查看该指令的历史调试运行记录，用户难以追溯过往执行情况。

**截图说明**：原调试页无运行记录相关入口或展示区域。

#### 问题四：操作日志缺失

**现状**：无法查看浏览器在调试过程中每一步的具体操作、目标元素、执行结果及截图，问题排查困难。

**截图说明**：原调试页仅有状态文字提示，无操作步骤明细。

#### 问题五：状态反馈不足

**现状**：调试运行时仅有简单的文字状态提示，缺少实时计时器和加载动画，用户无法感知任务是否在进行中。

**截图说明**：状态区域仅展示静态文字，无动态反馈。

#### 问题六：设置信息分散

**现状**：指令的基本配置信息（分组、权限、管理员等）没有集中管理入口，用户无法在调试页快速查看指令归属信息。

**截图说明**：指令元信息散落在不同位置或未展示，缺乏统一查看入口。

## 3. 优化方案详述

### 3.1 列表页优化

#### 3.1.1 分组化卡片布局

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| 组织方式 | 平铺网格卡片，无分组概念 | 按分组横向排列卡片，每个分组有独立标题行 |
| 卡片宽度 | 四列自适应，宽度不固定 | 固定宽度 260px，保证视觉一致性 |
| 浏览方式 | 纵向滚动浏览全部卡片 | 分组标题行 + 组内横向滚动浏览 |

**方案详情**：

- 指令按所属分组聚合展示，每个分组为一个独立区块。
- 分组标题行左侧带 4px 蓝色竖条指示器，右侧显示该分组下的指令数量（如「数据处理 (4)」）。
- 组内卡片横向排列，超出可视区域时通过水平滚动浏览，底部自定义细滚动条（高度 6px）。
- 卡片移除大面积占位图，改为左上角蓝色代码图标（28px × 28px），释放空间给文字内容。

**预期效果**：用户可按业务分组快速定位指令，组内横向浏览减少纵向滚动成本，信息展示更聚焦。

#### 3.1.2 搜索筛选增强

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| 筛选条件 | 仅名称搜索 | 名称搜索 + 分组下拉筛选 |
| 操作按钮 | 无查询/重置按钮 | 明确的「查询」「重置」按钮 |

**方案详情**：

- 搜索栏左侧放置「名称」输入框（带搜索图标）和「分组」下拉选择器（默认「全部」）。
- 搜索栏右侧放置「查询」（Primary 样式）和「重置」（白色边框样式）按钮。
- 「新增指令」和「体验中心」按钮统一置于搜索栏最右侧，与筛选操作区分。

**预期效果**：筛选交互更规范，用户可通过分组快速缩小范围，查询/重置按钮提供明确的操作反馈。

#### 3.1.3 卡片操作优化

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| 操作项 | 「编辑」「另存为模板」「删除」平铺 | 「设置」「调试」「更多(...)」 |
| 低频操作 | 与高频操作同级展示 | 收起至「更多」下拉菜单（复制、删除） |

**方案详情**：

- 卡片底部操作区改为三个入口：「设置」（齿轮图标）、「调试」（播放图标）、「更多...」（三点图标）。
- 「更多」按钮点击后弹出下拉菜单，包含「复制」和「删除」选项，删除项标红提示风险。
- 卡片整体增加 hover 效果：`translateY(-2px)` 上移 + 阴影加深，增强可点击感。
- 卡片描述文字限制最多两行，超出省略，保证卡片高度统一。

**预期效果**：高频操作（调试、设置）直接可见，低频操作（复制、删除）收归二级菜单，界面更清爽，操作层次清晰。

#### 3.1.4 新增设置弹窗

**方案详情**：

- 点击卡片「设置」按钮打开模态弹窗（宽度 520px），集中展示指令的基本信息。
- 弹窗采用**只读模式**，避免用户误操作修改配置：
  - **指令名称**：只读输入框
  - **所属分组**：只读文本展示
  - **管理员**：Tag 标签只读展示（蓝色背景，不可删除）
  - **权限组**：只读输入框
  - **部门**：只读输入框
- 弹窗底部仅保留「关闭」按钮（Primary 样式）。
- 弹窗通过 `z-index: 1500` 确保在调试页面也能正常展示。

**预期效果**：用户可在不离开当前页面的情况下快速查看指令元信息，只读模式避免误编辑，弹窗统一了信息查看入口。

### 3.2 调试页优化

#### 3.2.1 左侧编辑区精简

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| 编辑区内容 | 名称、Prompt、封面URL、变量表格混杂 | 聚焦为 Prompt 专用编辑区 |
| 封面URL | 直接展示在编辑区 | 移至设置弹窗，编辑区不再展示 |
| 变量表格 | 直接展示在编辑区下方 | 通过「变量」按钮打开模态框查看 |
| Prompt编辑器 | 高度被压缩 | textarea 最大化，高度 400px |

**方案详情**：

- 左侧区域合并为「指令详情」卡片，上方展示指令名称（只读）和沙箱类型切换（浏览器沙箱 / 代码沙箱）。
- 中间以分隔线划分，下方为「Prompt 内容」编辑区。
- Prompt textarea 宽度 100%，高度 400px，背景色 #F7F8FA，无边框，获得更大编辑面积。
- 新增「变量」和「清空」操作按钮，与沙箱切换并列置于 Prompt 标题行右侧。
- 调试运行时，textarea 上方显示黄色提示条：「⚠️ 执行过程中修改提示词将不影响当前调试」。

**预期效果**：调试页左侧彻底聚焦于 Prompt 编辑，移除非核心字段干扰，编辑体验大幅提升。

#### 3.2.2 标题栏操作按钮重组

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| 标题行 | 单一标题行混合所有信息 | 清晰分为两行布局 |
| 第一行 | — | 「指令详情」标题 + 运行记录按钮 + 保存按钮 |
| 第二行 | — | Prompt 标题 + 沙箱切换 + 变量/清空按钮 |

**方案详情**：

- 第一行左侧为「指令详情」标题（带蓝色竖条指示器），右侧为「运行记录」按钮和「保存」按钮。
- 第二行左侧为「Prompt 内容」标题，右侧为沙箱类型切换（浏览器沙箱 / 代码沙箱）、「变量」按钮、「清空」按钮。
- 调试按钮移至右侧「调试详情」面板标题栏，使用 Primary 样式突出显示。

**预期效果**：信息层级清晰，操作按钮按功能域分组，减少视觉混乱。

#### 3.2.3 运行详情面板增强

**方案详情**：

- 调试页采用左右两栏布局：左侧 58% 为编辑区，右侧 40% 为运行详情面板。
- 运行详情面板包含以下内容：
  - **面板标题栏**：「调试详情」标题 + 「运行记录」按钮 + 「调试」按钮（Primary 样式）。
  - **任务状态区**：动态状态标签（等待调试 / 运行中 / 执行完成）+ 实时计时器（mm:ss 格式，monospace 字体）+ 加载动画图标（旋转 spinner）。
    - 运行中：蓝色文字 + spinner
    - 执行完成：绿色文字 + 成功图标
  - **浏览器预览区**：仿 Chrome 浏览器界面，顶部深色标题栏包含红/黄/绿三色控制点、网站图标/标题、地址栏。内容区未开始调试时显示占位提示（浏览器图标 + 「浏览器实时画面」文字）。

**预期效果**：调试状态实时可感知，浏览器预览区域提供沉浸式运行监控体验。

#### 3.2.4 新增运行记录抽屉

**方案详情**：

- 点击调试详情面板的「运行记录」按钮，或调试页标题栏的「运行记录」按钮，从右侧滑出抽屉面板。
- 抽屉宽度 **950px**，占满屏幕高度，带 overlay 遮罩层。
- 抽屉内表格列结构如下：

| 列名 | 宽度 | 说明 |
|------|------|------|
| 发起人 | 自适应 | 运行任务的发起用户 |
| 创建时间 | 自适应 | YYYY-MM-DD HH:mm:ss |
| 运行结果的 id | 自适应 | 运行结果唯一标识 |
| prompt | 自适应 | 执行的 Prompt 内容摘要 |
| 状态 | 100px | 彩色标签：成功(绿) / 失败(红) / 运行中(蓝) |
| id | 自适应 | 运行记录 ID |
| 操作 | 140px | 「详情」和「操作日志」按钮 |

- 状态字段统一显示中文（成功 / 失败 / 运行中），使用对应颜色的标签样式。
- 抽屉底部带分页控件，显示总条数和页码按钮。
- 点击「操作日志」按钮打开操作日志弹窗。

**预期效果**：用户可随时查看指令的历史运行记录，状态一目了然，操作日志入口便捷。

#### 3.2.5 新增操作日志弹窗

**方案详情**：

- 从运行记录抽屉的「操作日志」按钮打开，弹窗尺寸 **1100px × 80vh**。
- 弹窗层级 `z-index: 3000`，确保覆盖抽屉和其他弹窗。
- 弹窗内容分为两部分：
  1. **总览统计区**：三列卡片式统计
     - 总步骤数（大字号数字）
     - 执行 Action 数
     - 执行时长（秒，monospace 字体）
  2. **操作日志表格**：

| 列名 | 宽度 | 说明 |
|------|------|------|
| 执行步骤 | 60px | 序号，居中 |
| 页面地址 | 120px | 当前页面 URL，可点击，溢出省略 |
| 执行目标 | 200px | 操作目标描述，最多两行省略 |
| 执行过程 | 240px | 详细过程说明，最多两行省略 |
| Action结果 | 320px | 成功/失败标签 + 结果描述卡片 |

- Action 结果区域内部以卡片形式展示每条结果，成功为绿色背景卡片，失败为红色背景卡片。
- 表格行 hover 背景变为 #F0F5FF。
- 点击截图缩略图可放大查看，放大遮罩层 `z-index: 4000`。

**预期效果**：完整呈现浏览器每一步操作的细节，便于问题回溯和排查，截图放大功能提升可读性。

### 3.3 视觉设计升级

#### 3.3.1 设计系统建立

| 层级 | 值 | 用途 |
|------|-----|------|
| 主色 | #4353FF | Primary 按钮、链接、状态指示器、图标高亮 |
| 主色悬停 | #3644E0 | 按钮 hover 状态 |
| 主色浅色 | #EEF0FF | Tag 背景、hover 背景 |
| 背景色 | #F5F7FA | 页面全局背景 |
| 卡片背景 | #FFFFFF | 卡片、弹窗、面板背景 |
| 边框色 | #E8E8E8 | 输入框边框、分割线 |
| 文字主色 | #1A1A2E | 标题、重要文字 |
| 文字次色 | #666666 | 描述、次要信息 |
| 文字三级 | #999999 | 占位符、时间、提示 |

**圆角等级**：

| 等级 | 值 | 用途 |
|------|-----|------|
| sm | 4px | 标签、小按钮、输入框 |
| md | 8px | 卡片、普通按钮、弹窗 |
| lg | 12px | 大弹窗容器 |
| xl | 20px | 特殊大圆角场景 |

**阴影等级**：

| 等级 | 值 | 用途 |
|------|-----|------|
| sm | 0 1px 3px rgba(0,0,0,0.05) | 默认卡片 |
| md | 0 4px 12px rgba(0,0,0,0.08) | 下拉菜单、hover 状态 |
| lg | 0 8px 24px rgba(0,0,0,0.12) | 弹窗、抽屉 |

#### 3.3.2 组件样式统一

**卡片**
- 白色背景 + `shadow-sm` + `radius-md`
- hover 时：`translateY(-2px)` + `shadow-lg`，过渡时间 0.25s

**按钮**
- **Primary**：主色背景 + 白色文字，hover 加深
- **Outline**：白色背景 + 主色边框，hover 主色浅色背景
- **White**：白色背景 + 灰色边框，hover 主色边框 + 主色文字

**标签**
- 成功：#F6FFED 背景 + #52C41A 文字 + #B7EB8F 边框
- 失败：#FFF2F0 背景 + #FF4D4F 文字 + #FFCCC7 边框
- 运行中：#E6F7FF 背景 + #4353FF 文字 + #91D5FF 边框

**弹窗/抽屉**
- overlay 遮罩：`rgba(0,0,0,0.5)`，带淡入动画
- 面板：`scale(0.95)` → `scale(1)` 放大入场，0.25s ease
- 抽屉：`translateX(100%)` → `translateX(0)` 侧滑入场，0.3s ease

**自定义滚动条**
- 高度 6px，圆角 3px
- 默认色：#E8E8E8，hover 色：#999999

## 4. 交互流程说明

### 4.1 列表页操作流程

**浏览与筛选**

1. 用户进入列表页，指令按分组横向排列展示。
2. 用户可在搜索栏输入名称关键词，选择分组下拉筛选器，点击「查询」执行筛选。
3. 点击「重置」清空筛选条件，恢复全部展示。

**卡片操作**

1. **调试**：点击卡片「调试」按钮，直接进入该指令的调试页面。
2. **设置**：点击卡片「设置」按钮，打开只读设置弹窗，查看指令元信息，点击「关闭」关闭弹窗。
3. **更多操作**：点击「更多(...)」，下拉菜单展示「复制」和「删除」。
   - 点击「复制」打开复制弹窗，输入新名称后确认。
   - 点击「删除」打开确认对话框，确认后删除指令。

### 4.2 调试页操作流程

**进入调试页**

1. 用户从列表页点击「调试」按钮，进入调试页面。
2. 页面展示面包屑导航：模版中心 / [指令名称]调试页面。

**Prompt 编辑与调试**

1. 用户在左侧 Prompt textarea 中编辑内容。
2. 点击「变量」按钮打开变量设置弹窗（系统变量 + 自定义变量 Tab），查看或编辑变量。
3. 选择沙箱类型（浏览器沙箱 / 代码沙箱）。
4. 点击右侧「调试」按钮启动调试，任务状态区展示「运行中」+ 计时器 + spinner。
5. 调试期间 textarea 上方显示黄色警告提示：「执行过程中修改提示词将不影响当前调试」。
6. 调试完成后，状态区展示「执行完成」+ 总耗时，浏览器预览区展示运行结果。

**查看运行记录**

1. 点击「运行记录」按钮，右侧滑出运行记录抽屉。
2. 抽屉内表格展示历史运行记录，每行包含状态标签和操作按钮。
3. 点击「详情」查看运行详情；点击「操作日志」打开操作日志弹窗。
4. 点击抽屉右上角「×」或 overlay 遮罩关闭抽屉。

**查看操作日志**

1. 从运行记录抽屉点击「操作日志」打开弹窗。
2. 弹窗顶部展示总览统计（总步骤数、Action 数、执行时长）。
3. 下方表格展示每一步操作的页面地址、目标、过程、结果。
4. 点击截图缩略图可放大查看，点击遮罩或关闭按钮退出放大视图。
5. 点击弹窗右上角「×」关闭操作日志弹窗。

**保存与设置**

1. 点击「保存」按钮保存当前 Prompt 修改，显示 Toast 提示「保存成功」。
2. 点击标题栏齿轮图标打开设置弹窗（只读），查看指令配置信息。

### 4.3 弹窗层级规范

| 层级 | z-index | 组件 |
|------|---------|------|

## 5. 设计规范

### 5.1 色彩规范

已在 3.3.1 设计系统中完整定义，开发时请严格使用 CSS 变量，避免硬编码色值。

### 5.2 字体规范

- 全局字体栈：`-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif`
- 计时器/代码字体：`'SF Mono', Monaco, monospace`

### 5.3 间距规范

- 页面内边距：24px
- 卡片内边距：16px ~ 20px
- 卡片间距：16px（组内横向），24px（分组纵向）
- 按钮内边距：8px 16px（标准），6px 12px（小型）

### 5.4 动画规范

| 场景 | 时长 | 缓动函数 |
|------|------|----------|
| 按钮 hover | 0.15s | ease |
| 卡片 hover 阴影/位移 | 0.25s | ease |
| 弹窗淡入缩放 | 0.25s | ease |
| 抽屉侧滑 | 0.3s | ease |
| Toast 滑入 | 0.3s | ease |
| 图片放大 | 0.3s | ease |
