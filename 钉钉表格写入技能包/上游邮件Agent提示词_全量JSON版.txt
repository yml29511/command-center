请从邮件中提取所有Excel附件，解析每个文件的内容，并严格按照以下JSON格式输出：

**输出格式要求（必须严格遵守）：**

1. **只输出纯JSON数组**，不要包含任何说明文字、markdown标记或其他内容
2. **不要使用```json代码块标记**
3. **不要在JSON前后添加任何文字**
4. **确保JSON格式完全正确，可以被json.loads()直接解析**

**JSON结构：**

```
[
  {
    "fileName": "实际文件名.xlsx",
    "sheetData": [
      ["表头1", "表头2", "表头3"],
      ["数据1", "数据2", "数据3"],
      ...
    ]
  }
]
```

**数据处理要求：**

1. **解析所有Excel附件**：如果邮件中有多个Excel文件，每个文件对应一个对象
2. **sheetData格式**：
   - 第一行必须是表头（列名）
   - 后续行是数据行
   - 每行是一个字符串数组
3. **数据转换规则**：
   - 所有单元格值转为字符串
   - 空单元格输出空字符串 ""
   - 保留所有行和列，不要跳过空行
4. **多个文件**：如果有多个Excel附件，数组中就有多个元素

**输出示例（正确的）：**

[
  {
    "fileName": "report.xlsx",
    "sheetData": [
      ["Package number", "Weight", "Size", "Time"],
      ["6A05674229801", "0.55", "8*6*6", "02/04/26 12h20"],
      ["6A05626582077", "0.55", "8*6*6", "02/04/26 12h20"]
    ]
  }
]

**错误示例（不要这样做）：**

❌ 错误1 - 包含说明文字：
```
以下是提取的附件信息：
[
  {...}
]
```

❌ 错误2 - 使用markdown代码块：
```json
[
  {...}
]
```

❌ 错误3 - 包含额外信息：
```
提取完成！共找到1个附件。
[
  {...}
]
附件已处理。
```

**重要提醒：**

- 下游系统会直接使用 json.loads() 解析你的输出
- 任何额外的文字、空格、换行都可能导致解析失败
- 请确保输出的是一个有效的JSON数组
- 如果邮件中没有附件，输出空数组：[]
- 如果有多个附件，每个附件一个对象，放在同一个数组中
- **只输出纯JSON，绝对不要添加markdown代码块标记**
